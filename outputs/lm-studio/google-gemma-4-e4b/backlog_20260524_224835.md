**User Story:** As a registered user, I want to log in using my email and password so that I can access personalized features of the application.

**Development Task:**

- Task ID: TASK-001
- Task Title: Implement JWT authentication endpoint
- Related User Story: US-001 (Login)
- Description: Create a new `/api/auth/login` POST endpoint. This endpoint must accept `email` and `password`. The backend logic must validate credentials against the user database, hash the password using bcrypt, and upon successful validation, generate and return a JSON Web Token (JWT) containing the User ID and expiration time.
- Acceptance Criteria:
    1. Successful login returns a 200 status code and a valid JWT in the response body.
    2. Invalid credentials (wrong password/email) return a 401 Unauthorized status code.
    3. The endpoint must utilize environment variables for secret key management.
- Estimated Effort: 5h, 3 story points
- Dependencies: None

**User Story:** As an administrator, I want to view a list of all active users and their associated account details so that I can monitor system usage.

**Development Task:**

- Task ID: TASK-002
- Task Title: Build User Listing API endpoint and component
- Related User Story: US-002 (Admin View Users)
- Description: Develop a GET request handler at `/api/admin/users` that queries the database for all users marked as 'active'. The response payload must include `user_id`, `email`, `registration_date`, and `last_login`. Additionally, create a dedicated React component (`UserTable.js`) to consume this endpoint and display the data in a paginated, sortable table format.

    1. The API returns an array of user objects with all required fields upon successful request (200 OK).
    2. The frontend component correctly handles pagination limits (e.g., 25 users per page) and displays loading states while fetching data.
    3. Access to this endpoint must be restricted via middleware, requiring an 'admin' role JWT token.
- Estimated Effort: 8h, 5 story points
- Dependencies: TASK-001 (Authentication mechanism must be in place for role checking)

**User Story:** As a user, I want to reset my password via a secure link sent to my registered email address so that I can regain access if I forget my credentials.

**Development Task:**

- Task ID: TASK-003
- Task Title: Implement Password Reset Token Generation and Email Service
- Related User Story: US-003 (Password Recovery)
- Description: Create a new workflow starting with a `/api/auth/forgot-password` endpoint that accepts only the user's email. The backend must generate a unique, time-limited token (e.g., 1 hour expiry), store it in the database linked to the user ID, and trigger an email service call (using SendGrid or similar) containing a secure reset link (`/reset?token=...`).
- Acceptance Criteria:
    1. The endpoint successfully generates and stores the token without exposing the token value in the response body.
    2. A testable mock of the email sending service is implemented, confirming that the correct URL structure is passed to the mailer function.
    3. The system must prevent brute-force attempts on this endpoint by implementing rate limiting (e.g., 5 requests per IP every 15 minutes).
- Estimated Effort: 6h, 4 story points
- Dependencies: None
